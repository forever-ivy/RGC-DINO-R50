## 二、技术复盘：为什么2-fold融合失败？

### 2.1 验证集 vs 测试集的鸿沟

**验证集表现**：
```
fold0: 0.3279 (强)
fold2: 0.2872 (次强，-12.4%)
```

**测试集推测**：
```
fold0: 45.044 (实测)
fold2: 可能<40分 (未单独测试，推测比验证集差距更大)
```

**关键问题**：
- fold2在验证集上看起来"还行"(0.2872)
- 但测试集上可能性能崩盘
- 验证集和测试集**分布不一致**

**教训**：
> **验证集表现不能预测测试集表现**
> 
> 尤其是：
> 1. 数据量小（2000样本）
> 2. 存在fold划分不均衡（fold1验证集47%异常）
> 3. 测试集可能有domain shift

---

### 2.2 Fold质量差异被低估

**训练数据量差异**：
```
fold0: 1463训练样本 → mAP 0.3279 → 测试集45.044
fold2: 1481训练样本 → mAP 0.2872 → 测试集可能<40
```

**问题分析**：
- 训练样本数量相近
- 但fold2性能明显更差（-12.4%）
- 说明**不是样本量问题，是样本质量/分布问题**

**可能原因**：
1. fold2的验证集包含了"简单样本"，训练集是"困难样本"
2. fold2的训练集domain与测试集不匹配
3. Grouped stratified 3-fold划分策略有问题

**教训**：
> **Fold划分质量比Fold数量更重要**
> 
> 宁愿用1个高质量fold，不用3个低质量fold

---

### 2.3 NMS融合的局限性

**我们的融合策略**：
```python
# fuse_multifold_predictions.py
score = max([b[5] for b in cluster])  # NMS：保留最高分
```

**理论**：
- NMS保留最高分，避免了TTA朴素平均的问题 ✓
- 多模型检出应该提升recall ✓

**现实**：
- fold2引入的框大部分是**低质量误检**
- 虽然NMS保留了最高分，但：
  - fold0和fold2检出的框**空间位置不同**
  - fold2的误检框不会被fold0覆盖
  - 最终提交的框数增加，但precision下降

**数据验证**（推测）：
```
fold0单独：
  - 高精度框：1000个样本 × 平均60个框 = 60,000框
  - Precision高，但recall可能不满

2-fold融合：
  - fold0: 60,000高质量框
  - fold2: 新增20,000低质量框
  - 总计80,000框，但新增的都是噪声
  - Precision下降 > Recall提升 → mAP下降
```

**教训**：
> **融合不一定提升，弱模型会拉低整体**
> 
> 当第二个模型比第一个模型弱超过10%时，融合往往是负收益

---

### 2.4 Top-100截断的影响

**提交限制**：每张图最多100个框

**融合后的影响**：
```
fold0单独：
  - 大部分图<100框
  - 只有54张图超过100框需要截断
  
2-fold融合：
  - 融合后框数增加
  - 更多图超过100框
  - 截断时可能丢失fold0的高质量框
```

**举例**：
```
某张图：
  fold0: 80个框（都是高质量）
  fold2: 60个框（大部分低质量）
  融合后: 120个框 → 截断到100
  
  可能的问题：
  - 如果fold2有20个框score > fold0的最后20个框
  - 截断会丢失fold0的20个真阳性
  - 保留fold2的20个假阳性
  - 结果：recall下降，precision也下降
```

**教训**：
> **Top-100截断会放大弱模型的负面影响**

## 三、根本原因分析

### 3.1 主要原因（按重要性排序）

#### 1. **Fold2质量远低于预期**（权重40%）

**证据**：
- 验证集：fold2比fold0差12.4%
- 测试集：fold2可能比fold0差>20%
- 融合后不升反降

**原因**：
- fold2的训练集与测试集domain不匹配
- 或者fold2包含了大量困难样本，导致泛化能力差

**应该做什么**：
- 在融合前，先在测试集上单独测试fold2
- 如果fold2 < 43分，直接放弃融合

---

#### 2. **验证集不能代表测试集**（权重30%）

**证据**：
- fold0验证集0.3279 → 测试集45.044（符合预期）
- fold2验证集0.2872 → 测试集可能<40（验证集高估）

**原因**：
- 2000样本太少，划分后验证集更少（500左右）
- 验证集和测试集domain不一致
- Grouped stratified划分可能引入偏差

**应该做什么**：
- 用测试集伪标签做validation（半监督）
- 或者用交叉验证多次验证

---

#### 3. **NMS融合无法过滤弱模型噪声**（权重20%）

**证据**：
- NMS保留最高分，但无法删除fold2的独有误检
- 融合增加了总框数，但precision下降

**原因**：
- NMS只能在同一位置的框中选最高分
- 不同位置的框都会保留
- 弱模型的误检大多在不同位置

**应该做什么**：
- 用WBF代替NMS（加权融合）
- 或者用voting（只保留多个模型都检出的框）

---

#### 4. **缺少融合前的A/B测试**（权重10%）

**我们的错误**：
- 没有在验证集上测试融合效果
- 直接提交测试集
- 盲目相信"融合一定提升"

**应该做什么**：
- 在验证集上先验证融合策略
- 确认提升后再提交测试集

---

### 3.2 次要原因

1. **Fold1异常未修复**：
   - fold1验证集占47%，训练集太小
   - 如果用了3-fold融合可能更差

2. **训练epoch不足**：
   - 只训练12 epochs
   - fold2可能还没收敛

3. **缺少后处理调优**：
   - NMS阈值、confidence阈值都是默认值
   - 没有针对融合场景优化

## 五、经验教训总结

### 5.1 关于模型融合

**✓ 正确的融合前提**：
1. **所有模型质量接近**（差距<5%）
2. **所有模型都在验证集上验证过**
3. **融合策略在验证集上验证过**
4. **弱模型不如不融合**

**✗ 错误的假设**：
1. ~~"融合一定比单模型好"~~ → **错误**
2. ~~"验证集好就是测试集好"~~ → **错误**
3. ~~"NMS能解决所有问题"~~ → **错误**
4. ~~"多个fold一定比一个fold好"~~ → **错误**

---

### 5.2 关于Fold策略

**教训1：Fold质量>Fold数量**
```
1个高质量fold (45分) > 2个中等质量fold融合 (44分)
```

**教训2：必须在测试集上验证每个fold**
```
验证集好 ≠ 测试集好
必须单独提交每个fold到测试集才知道真实性能
```

**教训3：Fold划分要严格均衡**
```
fold1验证集47% → 异常
fold0验证集27% → 正常
fold2验证集26% → 正常

异常fold必须重新划分
```

---

### 5.3 关于验证策略

**当前错误流程**：
```
训练 → 验证集评估 → 直接提交测试集 → 失败
```

**正确流程应该是**：
```
训练 → 验证集评估 → 
       ↓
       单个fold测试集提交（验证质量）→
       ↓
       确认所有fold都>baseline →
       ↓
       验证集测试融合策略 →
       ↓
       测试集融合提交
```

**关键点**：
- **每个fold必须在测试集单独验证**
- **融合必须在验证集先测试**
- **只有确认提升才提交测试集**

---

### 5.4 关于融合策略

**NMS的局限性**：
```python
# NMS只能处理重叠框
# 无法处理：
# 1. 弱模型的独有误检（不重叠）
# 2. 弱模型的低分真阳性被过滤
# 3. 模型质量差异导致的偏差
```

**更好的策略**：

**方案A：WBF（Weighted Boxes Fusion）**
```python
# 考虑模型权重
score = weighted_average(scores, weights=[0.7, 0.3])  
# fold0权重0.7，fold2权重0.3
```

**方案B：Voting**
```python
# 只保留多数模型都检出的框
if num_detections >= 2:  # 至少2个模型检出
    keep_box()
```

**方案C：Confidence-weighted fusion**
```python
# 根据验证集表现动态调整权重
weight_fold0 = 0.3279 / (0.3279 + 0.2872) = 0.53
weight_fold2 = 0.2872 / (0.3279 + 0.2872) = 0.47
```
